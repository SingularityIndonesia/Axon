# Axon Core

The runtime library for the Axon framework. Contains all public API types — no code generation,
no reflection, no platform dependencies beyond `kotlinx-coroutines`.

---

## API

### `Intent<out R>`

Pure, immutable declaration of what a user wants to accomplish. Carries only input data — never
the result. The result type `R` is a compile-time contract only.

```kotlin
sealed class MyAppIntent<out R> : Intent<R>() {

    data class LoginIntent(
        val username: String,
        val password: String
    ) : MyAppIntent<LoginIntent.LoginResult>() {
        sealed class LoginResult {
            data class LoginSuccess(val token: String) : LoginResult()
            data object LoginBlocked : LoginResult()
            data object UnableToProcess : LoginResult()
        }
    }
}
```

Built-in fields:
- `createdAt: Long` — epoch milliseconds at construction time
- `parent: Intent<*>?` — the intent that spawned this one, if any

---

### `Resolver<I, R>`

Processing contract for a specific intent type. Implemented by resolver classes.

```kotlin
interface Resolver<I : Intent<R>, R> {
    suspend fun resolve(intent: I): R
}
```

Rules:
- Must **never throw** — all error states must be expressed through `R`
- May be a suspend function with real async work, or return immediately — both are valid
- Thread affinity is controlled by the caller's coroutine dispatcher, not by the resolver

---

### `@Resolve(intentClass)`

Marks a class as the resolver for a specific intent. Used by the KSP processor to auto-generate
`Axon.init()`. Without KSP, resolvers are registered manually via `Axon.registerResolver`.

```kotlin
@Resolve(LoginIntent::class)
class LoginResolver @Inject constructor(
    private val authService: AuthenticationService
) : Resolver<LoginIntent, LoginResult> {
    override suspend fun resolve(intent: LoginIntent): LoginResult {
        val token = authService.login(intent.username, intent.password)
        return LoginSuccess(token = token)
    }
}
```

---

### `@Inject`

Marks a primary constructor as injectable by the KSP processor. The processor resolves the
full dependency graph at compile time — no runtime reflection required.

Classes with a no-arg constructor are injectable without `@Inject`.

```kotlin
class AuthenticationService @Inject constructor(
    private val api: AuthWebApi,
    private val db: LocalDatabase
)
```

---

### `Axon`

Central dispatcher. Routes intents to their registered resolvers and returns the result.
Designed to live as a singleton via DI — not a Kotlin `object`, so it stays testable.

```kotlin
val axon = Axon()
axon.init() // generated by KSP

val result = axon.dispatch(LoginIntent(username = "steve", password = "secret"))
```

Exceptions thrown by `dispatch`:
- `NoHandlerException` — no resolver registered for this intent type
- `ResolverException` — resolver violated its contract by throwing (always a bug)

An `AxonLogger` can be injected to route fatal errors to Crashlytics, Timber, Sentry, etc.:

```kotlin
val axon = Axon(logger = MyLogger())
```
